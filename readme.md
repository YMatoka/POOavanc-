ce qu'est l'injection de dépendence:

- Cela permet aux développeurs d'avoir une structure de code respectant le principe SOLID,un code structuré , facile a testé et amélioré.
Le principe est de permettre de modifier la classe plus facilement sans incidence sur le reste du code, en séparent
les fonctions de la classe.


ce qu'est le design pattern observer:
- C'est est une façon de coder qui s'effectue en respectant le contexte ou le sujet d'un projet.


ça permet une clarté  et une facilité de codage supplémentaire.

ce qu'est le temporal coupling:
Cela consiste à faire appel à des éléments d'une classe dans un ordre bien défini.
afin d'améliorer leur communication.
